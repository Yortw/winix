# qr — Design

**Date:** 2026-04-20
**Tool:** `qr`
**Release:** v0.5.0 (TBD)
**Status:** Approved

## Goal

`qr` is a CLI that turns text into QR codes: terminal-unicode by default on a TTY, SVG by default when stdout is redirected, PNG or ASCII on demand. URI-style helper subcommands (`wifi`, `sms`, `mailto`, `geo`, `tel`) cover the common non-text payloads. Single AOT native binary. Ships as a Winix suite member with the usual install/discoverability surface.

## Positioning

Online QR generators are a privacy hole: pasting a Wi-Fi password, a crypto address, or an internal URL into a web form leaks it. `qr` closes that gap with a fully offline generator that:

- **Is already cross-platform** — no shell-script-plus-`qrencode`-plus-ImageMagick incantation, no Python one-liner, no `pip install qrcode`.
- **Defaults sanely by context** — TTY gets a scannable unicode-art QR; a pipe gets SVG that won't corrupt a terminal.
- **Bundles the common non-text helpers** — Wi-Fi, SMS, mailto, geo, tel — as first-class subcommands rather than "you figure out the payload format".
- **Stays composable** — reads positional or stdin, writes stdout or `--output`, integrates with other Winix tools (e.g., `qr "$(url build …)"`).

**Cross-suite synergy:** `ids --type uuid7 | xargs -I{} qr "https://app.example.com/invite/{}"` — generate invite QRs in bulk. Pairs with `url build`, `clip`, `notify`, and (future) `http --qr` for LAN bind-URL display.

## CLI Interface

**Default (text payload):**

```
qr "https://example.com"       # positional
qr -                           # explicit stdin
cat payload.txt | qr           # bare stdin when no positional
```

**Helper subcommands:**

```
qr wifi   --ssid <s> [--password <p>] [--security wpa2|wpa|wep|nopass] [--hidden]
qr sms    --number <n> [--message <m>]
qr mailto --to <a> [--subject <s>] [--body <b>] [--cc <a>] [--bcc <a>]
qr geo    --lat <n> --lon <n> [--query <q>]
qr tel    --number <n>
```

**Global flags:**

| Flag | Default | Description |
|---|---|---|
| `--format {unicode,ascii,svg,png}` | `auto` | `unicode` on TTY; `svg` when stdout is redirected. |
| `--size N` / `-s` | `10` | Pixels per module (PNG/SVG only; ignored by unicode/ascii). |
| `--error-correction {l,m,q,h}` / `-e` | `m` | QR error correction level. |
| `--no-margin` | off | Strip the 4-module quiet zone. |
| `--output PATH` / `-o` | stdout | Write to file instead of stdout. |
| `--force-binary` | off | Allow PNG output to a TTY (otherwise refused). |
| Standard | | `--describe`, `--help`, `--version`, `--color`, `--no-color`. |

**Positional + stdin:**

- The default (text) subcommand takes a single positional argument: `qr "payload"`.
- A bare `-` positional reads stdin as a single UTF-8 string (trailing newline stripped).
- If no positional is supplied and stdin is not a TTY, stdin is read automatically.
- Helper subcommands take no positional — all payload fields come from flags.
- No batch mode (one QR per invocation).

**Examples:**

```bash
qr "https://example.com"
# ▄▄▄▄▄▄▄ ▄▄▄ ▄ ▄▄▄▄▄▄▄
# █ ▄▄▄ █ ▄▄ █▄ █ ▄▄▄ █
# ... (unicode half-block QR)

qr "https://example.com" > code.svg
# SVG written to code.svg (format auto-detected as svg because stdout is redirected)

qr "https://example.com" --format png --output code.png
# PNG written to code.png

qr wifi --ssid HomeNet --password "s3cr3t" --security wpa2
# Unicode QR encoding "WIFI:T:WPA2;S:HomeNet;P:s3cr3t;;"

qr sms --number +15551234567 --message "Hello"
# Unicode QR encoding "sms:+15551234567?body=Hello"

qr mailto --to support@example.com --subject "Bug report"
# Unicode QR encoding "mailto:support@example.com?subject=Bug%20report"

qr geo --lat -41.2924 --lon 174.7787 --query "Wellington NZ"
# Unicode QR encoding "geo:-41.2924,174.7787?q=Wellington%20NZ"

qr tel --number +6441234567
# Unicode QR encoding "tel:+6441234567"

echo "long-payload-here" | qr --error-correction h
# Unicode QR at ECC H (30% recovery)
```

**Exit codes:**

| Code | Meaning |
|---|---|
| 0 | Success. |
| 125 | Usage error — bad flags, unknown subcommand, missing required value, empty payload, PNG to TTY without `--force-binary`. |
| 126 | Runtime error — payload exceeds QR capacity at the chosen ECC, invalid UTF-8 on stdin, invalid helper field value. |

## Architecture

Two class libraries plus the console app:

```
src/Winix.QrCode/                    — shared library (will be consumed by future http --qr)
  QrEncoder.cs                       — pure: (string, EccLevel) → QrMatrix
  QrMatrix.cs                        — record: size, bool[,] modules, quiet-zone width
  EccLevel.cs                        — enum: L, M, Q, H
  Renderers/
    UnicodeRenderer.cs               — pure: QrMatrix → string (half-block)
    AsciiRenderer.cs                 — pure: QrMatrix → string (full-block + space)
    SvgRenderer.cs                   — pure: QrMatrix → string
    PngRenderer.cs                   — pure: (payload, EccLevel, pixelsPerModule, drawQuietZone) → byte[]
                                       (asymmetric: QRCoder.PngByteQRCode consumes payload directly,
                                        not a pre-built matrix — documented below)

src/Winix.Qr/                        — tool-local class library
  QrOptions.cs                       — sealed record: SubCommand + payload + render flags
  SubCommand.cs                      — enum: Text, Wifi, Sms, Mailto, Geo, Tel
  OutputFormat.cs                    — enum: Auto, Unicode, Ascii, Svg, Png
  ArgParser.cs                       — argv → QrOptions, dispatches on subcommand
  PayloadBuilder.cs                  — pure: QrOptions → payload string
  Helpers/
    WifiPayload.cs                   — pure: fields → "WIFI:T:WPA2;S:…;P:…;H:true;;"
    SmsPayload.cs                    — pure: fields → "sms:+…?body=…"
    MailtoPayload.cs                 — pure: fields → "mailto:…?subject=…&body=…"
    GeoPayload.cs                    — pure: fields → "geo:lat,lon?q=…"
    TelPayload.cs                    — pure: fields → "tel:+…"
  OutputDispatcher.cs                — pure: (payload, EccLevel, OutputFormat, size, noMargin) → string | byte[]
                                       (routes to PngRenderer directly for Png; otherwise calls
                                        QrEncoder then the matching matrix-based renderer)
  Formatting.cs                      — error-message composition

src/qr/
  Program.cs                         — thin orchestrator
  qr.csproj                          — AOT, PackAsTool; PackageReference QRCoder 1.6.x
  README.md
  man/man1/qr.1

tests/Winix.QrCode.Tests/
  QrEncoderTests.cs
  UnicodeRendererTests.cs
  AsciiRendererTests.cs
  SvgRendererTests.cs
  PngRendererTests.cs

tests/Winix.Qr.Tests/
  ArgParserTests.cs
  PayloadBuilderTests.cs
  WifiPayloadTests.cs
  SmsPayloadTests.cs
  MailtoPayloadTests.cs
  GeoPayloadTests.cs
  TelPayloadTests.cs
  OutputDispatcherTests.cs
```

**Key design points:**

- **Shared library `Winix.QrCode` extracted from day one.** Future `http --qr` is the second consumer; extracting now sets the API boundary before a second caller starts making assumptions. `QrMatrix` is our own record — QRCoder types never leak past `QrEncoder`.
- **One file per pure logic unit.** Each renderer, each helper, each parser — ~40–100 lines, independently testable, no I/O.
- **Subcommand dispatch in `ArgParser`.** `ArgParser.Parse(argv)` returns `QrOptions` with a `SubCommand` enum and the appropriate fields populated. `Program.cs` switches on subcommand to call `PayloadBuilder`.
- **`PayloadBuilder` is the fan-in point.** Takes `QrOptions`, returns the string that goes to `QrEncoder`. Text mode is passthrough; helper modes call the corresponding `Helpers/*.cs` function.
- **TTY detection** uses `Yort.ShellKit.ConsoleEnv` — matches `treex`/`files`/`less` precedent.
- **Under the hood:** `QRCoder.QRCodeGenerator.CreateQrCode(payload, ECC)` produces a `QRCodeData` object. `QrEncoder` copies its module matrix into a `QrMatrix` record and discards the QRCoder reference. `QRCoder.PngByteQRCode` is used for PNG (no System.Drawing dependency, AOT-clean). SVG is hand-rolled (emit one `<path>` with all dark modules as a single `d` attribute — smaller output than per-rectangle). Unicode and ASCII renderers are pure string builders.
- **AOT:** QRCoder 1.6.x is AOT-safe; no reflection, no dynamic types. Trim-analyzer clean. `System.Uri.EscapeDataString` for helper percent-encoding is AOT-safe.

## Algorithm Choices

### Output Format Auto-Selection

```
--format auto (default) + stdout is TTY   → unicode
--format auto (default) + stdout piped    → svg
--format X                                 → X (explicit)
```

PNG to a TTY is refused with exit 125 unless `--force-binary` is given (binary on terminal corrupts the line editor).

### Unicode Rendering

Half-block characters pair two rows into one terminal cell:

| Top module | Bottom module | Glyph |
|---|---|---|
| dark | dark | `█` |
| dark | light | `▀` |
| light | dark | `▄` |
| light | light | ` ` |

Odd row count is padded with a blank row at the bottom. Includes a 4-module quiet zone unless `--no-margin`. Colour is default terminal fg/bg — no ANSI sequences emitted, so `NO_COLOR` is respected trivially.

### ASCII Rendering

Full-block: `##` (dark) + `  ` (light), two chars wide per module to preserve aspect ratio. One terminal row per QR module (vs two-modules-per-row for the unicode half-block path), so an ASCII QR takes roughly twice the vertical terminal space a unicode QR would. The trade-off is universal compatibility — works in terminals without unicode block-glyph fonts.

### SVG Rendering

Single `<path>` with every dark module contributing an `M x,y h N v N h -N z` segment, inside a `viewBox` sized `(modules + 2*quiet) × (modules + 2*quiet)`. The consumer sizes via CSS; `--size N` sets the nominal per-module pixel hint in width/height attributes.

### PNG Rendering

`QRCoder.PngByteQRCode` takes the payload string directly and produces `byte[]` via `GetGraphic(pixelsPerModule, darkRgba, lightRgba, drawQuietZone)`. This is the one renderer that does NOT take a `QrMatrix` — `PngByteQRCode` owns the whole encode-to-PNG pipeline internally. `PngRenderer` therefore takes `(payload, EccLevel, pixelsPerModule, drawQuietZone)` and calls QRCoder twice total across the process (once in `QrEncoder` for the text/SVG/unicode/ASCII path, once in `PngRenderer`). That's an acceptable ~1 ms overhead for the PNG-specific path and avoids re-implementing PNG chunk assembly to preserve a uniform renderer signature.

`drawQuietZone` flips on `--no-margin` (`drawQuietZone: false`). QRCoder's own quiet-zone handling means we don't have to pad the matrix ourselves on the PNG path.

### Helper Payload Formats

| Helper | Format |
|---|---|
| `wifi` | `WIFI:T:<security>;S:<ssid>;P:<password>;H:<hidden>;;` (de-facto ZXing spec). Escape `\`, `;`, `,`, `:`, `"` in SSID/password with a backslash prefix. `security` of `nopass` omits `P:` field. `hidden` omitted unless true. |
| `sms` | `sms:<number>?body=<percent-encoded-message>` (RFC 5724). |
| `mailto` | `mailto:<to>?subject=<enc>&body=<enc>&cc=<enc>&bcc=<enc>` (RFC 6068), omitted fields skipped. |
| `geo` | `geo:<lat>,<lon>?q=<percent-encoded-query>` (RFC 5870). Lat must be in `[-90, 90]`; lon in `[-180, 180]` — violations exit 126. |
| `tel` | `tel:<number>` (RFC 3966). No validation of the number string. |

Percent-encoding uses `System.Uri.EscapeDataString` (component-style, matches JavaScript `encodeURIComponent`).

### Error-Correction Default

Default `m` (15% recovery) matches `qrencode` and most library defaults. Users who need a smaller code use `-e l`; users printing for outdoor/damaged contexts use `-e q` or `-e h`.

## Edge Cases

- **Empty positional + empty stdin** → exit 125, `"qr: payload is empty"`.
- **Payload exceeds QR capacity at chosen ECC** — QR Model 1 max is 2,953 alphanumeric or 7,089 numeric chars at ECC L; much smaller at H. QRCoder throws; we catch and exit 126 with a message suggesting `-e l` or a shorter payload.
- **Non-UTF-8 bytes on stdin** — decoder replacement would silently corrupt; we detect and exit 126 with `"qr: stdin is not valid UTF-8"`.
- **Wi-Fi SSID with special characters** — escape `\`, `;`, `,`, `:`, `"` per ZXing Wi-Fi URI convention. Test covers each.
- **Mailto `body` with `&` or `=`** — percent-encoded. Test covers.
- **Geo lat/lon out of range** — exit 126 with the specific field and range.
- **Unicode rendering with terminal width < QR size** — QR is emitted regardless; the terminal wraps. Documented; no special handling.
- **PNG/SVG with `--no-margin`** — strip the quiet zone from the matrix before rendering. May reduce scannability; documented warning in README.
- **`--size 0` or negative** → exit 125, `"qr: --size must be a positive integer"`.
- **`--format png` to a TTY without `--force-binary`** → exit 125 with remediation hint (suggest `--output` or `--force-binary`).

## Error Handling

| Situation | Exit | Message |
|---|---|---|
| Unknown subcommand | 125 | `qr: unknown subcommand: <x>` |
| Unknown / malformed flag | 125 | `qr: <ShellKit's stock message>` |
| Missing required flag (helper) | 125 | `qr: <helper>: missing required --<flag>` |
| Empty payload | 125 | `qr: payload is empty` |
| PNG → TTY without `--force-binary` | 125 | `qr: refusing to write PNG to terminal (use --output or --force-binary)` |
| `--size` ≤ 0 | 125 | `qr: --size must be a positive integer` |
| Payload exceeds QR capacity | 126 | `qr: payload too long for ECC level <X>; try --error-correction l or shorten the payload` |
| Stdin is not valid UTF-8 | 126 | `qr: stdin is not valid UTF-8` |
| Helper field out of range / malformed | 126 | `qr: <helper>: <field>: <reason>` |
| Any other `Exception` | 1 | `qr: error: <message>` |

## Testing

Target ~110 tests.

- **`QrEncoderTests`** — encode short/medium/long payloads at each ECC level; verify `QrMatrix` size and module-count invariants; capacity-overflow throws; empty input throws; known-fixture hash check against a reference implementation (ZXing or a stored byte-exact expected matrix).
- **`UnicodeRendererTests`** — 2×2, 3×3, 21×21 (smallest real QR); odd/even row handling; quiet-zone on/off; byte-exact output.
- **`AsciiRendererTests`** — same shape suite as unicode.
- **`SvgRendererTests`** — SVG root element, viewBox, path `d` attribute; quiet-zone effect on viewBox; size attribute respects `--size`.
- **`PngRendererTests`** — PNG magic bytes present (`89 50 4E 47 0D 0A 1A 0A`); IHDR width/height parsed from bytes 16-23 match `(modules + 2*quiet) * size`; output length non-zero. Byte-level inspection only — no image-decode dependency, keeps tests cross-platform-safe (no System.Drawing in the test graph).
- **Helper payload tests** — each helper across happy-path and escape cases. Wi-Fi: semicolon/colon/backslash/comma in password; hidden true/false; `nopass` security. SMS: numbers with `+`, `-`, `(` `)`, `.`; message with emoji. Mailto: subject/body with `&`, `=`, spaces, unicode; multiple cc/bcc. Geo: lat/lon negative, out of range, query with spaces. Tel: numbers with international prefix.
- **`ArgParserTests`** — subcommand dispatch, flag propagation, helper-required-flag enforcement, unknown subcommand, `--format` parsing, `--error-correction` parsing, mutual exclusions.
- **`OutputDispatcherTests`** — format auto-selection via a faked `ConsoleEnv`, PNG-to-TTY refusal behaviour, `--force-binary` override.
- **`PayloadBuilderTests`** — subcommand → payload routing, text passthrough, helper fan-out.

## Distribution

Pipeline integration follows the url/notify/digest pattern:

- `bucket/qr.json` — scoop manifest
- `.github/workflows/release.yml` — pack/publish/zip/combined-zip/tools-map entries for `qr`
- `.github/workflows/post-publish.yml` — manifest update + winget generator entries
- `docs/ai/qr.md` + entry in `llms.txt`
- `src/qr/README.md` + `src/qr/man/man1/qr.1`
- `CLAUDE.md` updates: project layout, NuGet package IDs, scoop manifests list
- NuGet package ID: `Winix.Qr`
- Scoop binary: `qr.exe`
- Shared lib `Winix.QrCode` stays internal (not a public NuGet) — matches `Winix.FileWalk` / `Winix.Codec` precedent.
- **Name collision watch:** `qr` is also the script name installed by the Python `qrcode` package. Environments with both installed resolve by PATH order. README documents the potential shadowing.

## Out of Scope (v1)

- **Image-decode** (QR image → text). Deferred to v2 along with ZXing.Net AOT validation.
- **vCard / iCal helpers.** Multi-line records with escape rules and (for iCal) RFC 5545 timestamp/timezone semantics. Own brainstorm required.
- **Custom colours** (`--fg`, `--bg`), embedded logos, rounded-module styles. Cosmetic; add later if asked.
- **Batch mode** (multi-payload input). Shell loops cover it.
- **Micro-QR, Structured Append, Kanji mode.** QR Model 1 only.
- **Clipboard integration** (`--copy`). Compose with `clip`: `qr "…" --format svg | clip`.

## Open Implementation Questions (decide during plan)

- Whether to use `QRCoder.PayloadGenerator` helpers (ships Wi-Fi/SMS/mailto/geo/tel payload builders) or hand-roll. Hand-rolled chosen for control (ADR D6); this note is a reminder to confirm the API surface stays tool-local during plan.
- `QRCoder.PngByteQRCode` exact `GetGraphic` signature across the target QRCoder version (1.6.x). Documented above as `GetGraphic(pixelsPerModule, darkRgba, lightRgba, drawQuietZone)`; verify against the pinned package version during plan.
- Exact `ArgParser` subcommand-dispatch shape — ShellKit has no built-in subcommand support. Follow the `schedule` / `url` precedent (each subcommand's flag set is parsed separately after subcommand extraction).
- Whether `SvgRenderer` emits a single `<path>` (compact) or one `<rect>` per module (simpler). Decision: single `<path>` for smaller output, measured during plan.
