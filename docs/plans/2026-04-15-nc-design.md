# nc — Cross-Platform Netcat Replacement (Design)

**Status:** Approved — 2026-04-15
**Companion ADR:** [2026-04-15-nc-adr.md](2026-04-15-nc-adr.md)
**Target release:** v0.2.0

## Overview

`nc` is a cross-platform netcat replacement targeting the Winix audience: Windows-first developers and sysadmins who use `Test-NetConnection` for port checks and reach for `curl`/PowerShell scripts when they want a quick TCP send-and-receive. On Linux/macOS, `nc` exists but its forks (BSD, GNU, ncat) all behave slightly differently. `nc` (Winix) provides a consistent cross-platform implementation with the modern features people actually use (TLS) and skips the niche/legacy ones (hex dump, exec, SOCKS).

Its job is to be the obvious answer to:
- "Is this port open?"
- "Send these bytes over TCP/UDP and show me the response."
- "Listen for one connection on this port and capture what arrives."
- "Test this HTTPS endpoint with a hand-crafted request."

## Scope

### In scope (v0.2.0)

- TCP and UDP client (connect mode)
- TCP and UDP listen mode (single connection / single datagram, then exit)
- Port-open check (`-z` / `--check`) with single, range, list, or mixed port specifiers
- TLS for client connections (`--tls` / `--ssl`)
- IPv4 / IPv6 selection (`-4` / `-6`)
- Connection / idle timeouts
- File transfer (works "for free" via stdin/stdout piping)
- Standard Winix flags: `--json`, `--describe`, `--color`, `--no-color`, `--help`, `--version`

### Out of scope (deferred)

- Hex dump (`-o`) — niche; users with serious protocol-debugging needs reach for Wireshark/mitmproxy
- Source-port binding (`-p`) — rarely needed in practice
- Exec on connect (`-e`) — security minefield; modern netcats restrict or remove it
- SOCKS proxy (`-x`) — niche; SSH `-D` covers most cases
- Port forwarding / relay — that's `socat` or SSH `-L` territory
- Multi-connection listen (accept and serve concurrently) — adds significant complexity
- TLS for listen mode (server certificates) — requires cert management
- DTLS (TLS over UDP) — niche

## Architecture

Standard Winix split: thin console + testable class library + xUnit project.

```
src/Winix.NetCat/                       — class library
  NetCatOptions.cs                      — parsed/validated options struct
  NetCatClient.cs                       — TCP/UDP outbound connect + relay
  NetCatListener.cs                     — TCP/UDP inbound listen + relay
  PortChecker.cs                        — --check mode (open/closed/timeout)
  RelayPump.cs                          — bidirectional stdin↔socket copy
  TlsWrapper.cs                         — wraps a NetworkStream in SslStream
  PortRangeParser.cs                    — parses 80,443,8080-8090 syntax
  Formatting.cs                         — status messages, JSON output
  Winix.NetCat.csproj

src/nc/                                 — console app (thin entry point)
  Program.cs                            — arg parsing via Yort.ShellKit, dispatch
  README.md                             — install, usage, examples, options table
  man/man1/nc.1                         — generated from nc.1.md
  nc.1.md                               — markdown source for man page
  nc.csproj

tests/Winix.NetCat.Tests/               — xUnit
  OptionsParsingTests.cs
  PortRangeTests.cs
  FormattingTests.cs
  RelayPumpTests.cs                     — in-memory streams, no sockets
  ClientListenerRoundtripTests.cs       — loopback + ephemeral port
  PortCheckerTests.cs
  HalfCloseTests.cs
  TimeoutTests.cs
  CancellationTests.cs
  Winix.NetCat.Tests.csproj

docs/ai/nc.md                           — AI agent guide
bucket/nc.json                          — Scoop manifest
```

**No new shared library.** Networking code is small and self-contained — no reuse opportunity with the existing `Winix.WhoHolds` (different APIs entirely: WhoHolds queries the local OS connection table, `nc` opens its own sockets).

### Key types

- `NetCatOptions` — frozen struct of validated args (Host, Port(s), Mode, Protocol, UseTls, Timeout, AddressFamily, BindAddress, InsecureTls, Verbose, JsonOutput).
- `NetCatClient.RunAsync(options, stdin, stdout, stderr, ct)` — outbound mode entry point.
- `NetCatListener.RunAsync(options, stdin, stdout, stderr, ct)` — inbound mode entry point.
- `PortChecker.CheckAsync(options, stderr, ct)` → `PortCheckResult` array.
- `RelayPump.RunAsync(socket, stdin, stdout, ct)` — bidirectional copy with half-close handling.

## CLI Surface

### Modes (mutually exclusive)

| Mode | Default? | How invoked |
|------|----------|-------------|
| Connect (outbound) | Yes | `nc HOST PORT` (positional args alone) |
| Listen (inbound) | No | `--listen` / `-l` |
| Check (port probe) | No | `--check` / `-z` |

### Flags

| Long | Short | Description |
|------|-------|-------------|
| `--listen` | `-l` | Listen for one inbound connection |
| `--check` | `-z` | Check if port(s) are open, exit immediately |
| `--udp` | `-u` | Use UDP (default is TCP) |
| `--tls` | `--ssl` | Wrap TCP connection in TLS (client mode only) |
| `--insecure` | | Skip TLS certificate validation (warning emitted) |
| `--ipv4` | `-4` | Force IPv4 |
| `--ipv6` | `-6` | Force IPv6 |
| `--timeout SEC` | `-w SEC` | Connection / idle timeout (default: no I/O timeout, 10s for `--check`) |
| `--bind ADDR` | | Bind listener to specific interface (default: all interfaces) |
| `--no-shutdown` | | Don't half-close socket on stdin EOF (rare protocols that hate it) |
| `--verbose` | `-v` | Show closed/timeout ports too in `--check` mode |
| `--json` | | Emit JSON summary to stderr |
| `--describe` | | Auto-generated agent metadata |
| `--color` / `--no-color` | | Colour control |
| `--help` / `--version` | | Standard |

### Positional arguments

| Mode | Positionals |
|------|-------------|
| Connect (default) | `HOST PORT` |
| `--check` | `HOST PORT[,RANGE]` (single, list, range, or mixed) |
| `--listen` | `PORT` (use `--bind` to choose interface) |

### Port range syntax (check mode)

```
80                    # single
80-1000               # range (inclusive)
80,443,8080           # list
80-100,443,8080-8090  # mixed
```

### Validation rules

- Exactly one of {default, `--listen`, `--check`} — error if combined.
- `--bind` only valid with `--listen` — error otherwise.
- `--tls` not allowed with `--udp` (DTLS not in scope).
- `--tls` not allowed with `--listen` (no server-cert handling in v0.2.0).
- `--insecure` only valid with `--tls`.
- `--ipv4` and `--ipv6` mutually exclusive.
- `--no-shutdown` only valid in connect / listen modes.
- `--verbose` only meaningful in `--check` mode.
- Each `PORT` must be 1-65535. Ranges must have low ≤ high.

### Examples

```bash
# Port checks
nc -z target.com 443
nc --check db.internal 5432 --timeout 3
nc -z target.com 80,443,5432
nc -z target.com 1-1024 --json

# TCP client
nc target.com 80
echo "GET / HTTP/1.0" | nc target.com 80
nc target.com 80 < request.bin > response.bin    # file transfer

# TLS client
nc --tls api.example.com 443
nc --tls --insecure dev-server.local 8443

# UDP client
nc -u dnsserver 53 < query.bin
echo "<14>test message" | nc -u syslog.local 514

# Listen mode
nc -l 8080                                       # accept one TCP connection
nc --listen --udp 1234                           # accept one UDP datagram
nc --listen 8080 --bind 127.0.0.1                # bind localhost only

# IP family
nc -6 ipv6.google.com 80
nc -4 dual-stack.example.com 80
```

## I/O Lifecycle

### Connect mode (TCP)

1. **DNS + connect** with timeout (default: 10s). On failure → exit 1 (refused/unreachable) or 2 (timeout).
2. Once connected, run two relays **in parallel**:
   - `stdin → socket` (write loop)
   - `socket → stdout` (read loop)
3. **When stdin reaches EOF**: call `Socket.Shutdown(SocketShutdown.Send)` so the peer sees end-of-stream on its read side, but keep the read loop going.
4. **When the socket's read returns 0 bytes** (peer closed): stop the read loop.
5. **When both relays have completed** → exit 0.
6. **On Ctrl-C**: cancel both relays, close socket, exit 130.

The half-close in step 3 is the critical detail. Without it, `echo "GET / HTTP/1.0\r\n\r\n" | nc target 80` hangs because the server is waiting for more request bytes. `--no-shutdown` is provided as an escape hatch for the rare protocols that interpret half-close as a full close.

### Connect mode (UDP)

Detection is via `Console.IsInputRedirected`:

- **Redirected stdin** (piped or from a file): read stdin, send each buffer-read as a datagram, then optionally wait briefly for a response (`--timeout SEC` after send, default 0 = exit immediately after sending).
- **Interactive stdin** (TTY): send each line as a datagram, print received datagrams to stdout, run until Ctrl-C.
- Datagram size limit: 65507 bytes (UDP max payload). Larger reads are split into multiple datagrams with a stderr warning per-split, since a single read isn't generally a "message" in UDP terms.

### Listen mode (TCP, single connection)

1. Bind to `(--bind, PORT)`. Default bind is `IPAddress.Any` (or `IPv6Any` if `-6`). Bind failure → exit 1 or 126 (perm denied).
2. Accept **one** connection. Optional `--timeout` limits how long to wait.
3. Once connected, identical relay behaviour to Connect mode (with half-close).
4. After connection completes, exit 0.
5. Subsequent connection attempts during the active session are refused by the OS.

### Listen mode (UDP)

1. Bind socket.
2. Receive one datagram (or until `--timeout` expires).
3. Print payload to stdout.
4. Exit.

### Check mode (`-z` / `--check`)

1. Resolve DNS once (cached for the run).
2. For each port in the range/list, fan out concurrent connection attempts (up to 32 in flight).
3. Each attempt: TCP `connect` only, immediate close on success.
4. Per-port results → output as designed below.
5. Exit code = "worst result wins" (timeout > closed > open).

### TLS (client only, v0.2.0)

- After TCP connect, wrap the `NetworkStream` in `SslStream`.
- `AuthenticateAsClientAsync(host)` — automatically sends SNI.
- Default: validate certificate chain. `--insecure` skips validation (with a stderr warning).
- Once authenticated, the relay treats the `SslStream` as the read/write target — neither relay direction is aware TLS is underneath.

### Cancellation discipline

A single `CancellationTokenSource` per run:
- `Console.CancelKeyPress` → cancel
- Idle / connect timeout → cancel
- One-side EOF → close that direction only, don't cancel the other

## Output, Exit Codes, JSON

### Stdout vs stderr

| Output | Stream | When |
|--------|--------|------|
| Bytes received from socket (connect/listen) | stdout | Always — this is the data conduit |
| Open ports (check mode, default) | stdout | One per line, suitable for piping |
| TLS warnings, status messages | stderr | E.g., "warning: skipping cert validation" |
| Connection errors, bind failures | stderr | Always |
| JSON summary (with `--json`) | stderr | Always |
| Closed/timeout ports (check mode) | stderr | Only with `-v` |

### Exit codes

| Code | Meaning |
|------|---------|
| 0 | Success — connection completed; all probed ports open; listener served one connection |
| 1 | Connection refused, host unreachable, DNS failure, bind failure, any closed port in check mode, TLS handshake failure |
| 2 | Timeout (connect or check) |
| 125 | Usage error (bad arguments) |
| 126 | Permission denied (e.g., privileged port) |
| 130 | Interrupted (Ctrl-C) |

### JSON schemas

Common envelope: `{tool, version, mode, exit_code, exit_reason, ...mode-specific...}`.

**Connect:**
```json
{
  "tool": "nc", "version": "0.2.0", "mode": "connect",
  "host": "target.com", "port": 80, "protocol": "tcp", "tls": false,
  "address_family": "ipv4", "remote_address": "93.184.216.34",
  "bytes_sent": 42, "bytes_received": 1305, "duration_ms": 187,
  "exit_code": 0, "exit_reason": "success"
}
```

**Listen:** same as connect plus `"local_address": "0.0.0.0:8080"`; remote becomes the connecting peer.

**Check:**
```json
{
  "tool": "nc", "version": "0.2.0", "mode": "check",
  "host": "target.com",
  "ports": [
    {"port": 80, "status": "open", "latency_ms": 23},
    {"port": 443, "status": "open", "latency_ms": 24},
    {"port": 5432, "status": "closed"}
  ],
  "exit_code": 1, "exit_reason": "some_closed"
}
```

`exit_reason` for check mode ∈ `{all_open, some_closed, some_timeout, all_failed}`.

### Colour

- Status messages on stderr use colour: green = open, red = closed/error, yellow = timeout/warning.
- Bytes relayed on stdout are **never coloured** — that data is going somewhere else (file, pipe).
- Respects `NO_COLOR` env var and `--color`/`--no-color` flags via `Yort.ShellKit.ConsoleEnv`.

## Error Handling

Principle: every error category produces a single descriptive line on stderr and a meaningful exit code. No stack traces.

### Network errors

| Error | User message (stderr) | Exit |
|-------|------------------------|------|
| DNS failure | `nc: cannot resolve "target.com"` | 1 |
| Connection refused | `nc: target.com:80 — connection refused` | 1 |
| Network unreachable | `nc: target.com:80 — host unreachable` | 1 |
| Connect timeout | `nc: target.com:80 — connection timed out after 10s` | 2 |
| Connection reset mid-stream | `nc: connection reset by peer` (after partial relay) | 1 |

### TLS errors

| Error | User message | Exit |
|-------|--------------|------|
| Cert chain invalid | `nc: TLS — certificate validation failed: <reason>` | 1 |
| SNI / wrong host | (same wrapped reason) | 1 |
| Protocol mismatch | `nc: TLS — handshake failed: <reason>` | 1 |
| `--insecure` used | (Warning, not error) `nc: warning — TLS certificate validation disabled` | (depends on connection result) |

### Bind errors (listen mode)

| Error | Message | Exit |
|-------|---------|------|
| Port in use | `nc: cannot bind :8080 — address already in use` | 1 |
| Permission denied | `nc: cannot bind :80 — permission denied (privileged port)` | 126 |
| Bad bind address | `nc: invalid bind address "10.99.99.99"` | 125 |

### I/O errors

| Error | Behaviour |
|-------|-----------|
| `BrokenPipeException` on stdout (e.g., `nc target 80 \| head`) | Treat as clean termination, exit 0 (POSIX-friendly) |
| Stdout disk full | `nc: write to stdout failed: <reason>`, exit 1 |
| Stdin read failure | Treat as EOF (graceful), don't raise |

### Argument errors

Caught by the `Yort.ShellKit` parser before any I/O. Produces standard help-text output and exit 125.

| Error | Message |
|-------|---------|
| Invalid port | `nc: invalid port "99999" — must be 1-65535` |
| Mode conflict | `nc: --listen and --check cannot be used together` |
| TLS + UDP | `nc: --tls is not supported with --udp` |
| TLS + listen | `nc: --tls is not supported with --listen (server certs are not supported in this version)` |
| `--bind` without `--listen` | `nc: --bind requires --listen` |

### Cancellation (Ctrl-C)

- During connect: cancel cleanly, message `nc: interrupted`, exit 130.
- During relay: shut down both directions, flush stdout, exit 130. Bytes already arrived are kept.
- During multi-port check: report partial results to stdout/JSON, exit 130.

## Testing Strategy

`tests/Winix.NetCat.Tests/` — xUnit, runs on Windows + Linux + macOS in CI.

### Unit tests (fast, no sockets)

| File | Scope |
|------|-------|
| `OptionsParsingTests.cs` | All argument combinations, validation rules, mutual exclusion |
| `PortRangeTests.cs` | Range parser (`80-100,443`) — edges, empty, overlapping, descending, invalid |
| `FormattingTests.cs` | Stderr status messages, JSON schema for each mode, colour vs no-colour |
| `RelayPumpTests.cs` | Bidirectional copy with in-memory streams (no sockets) — including stdin-EOF triggers half-close, both-EOF terminates, cancellation interrupts |

### Integration tests (real sockets, loopback only)

| File | Scope |
|------|-------|
| `ClientListenerRoundtripTests.cs` | `NetCatListener` + `NetCatClient` in same process, ephemeral port, bytes both ways |
| `PortCheckerTests.cs` | Bind a listener → check shows open. Don't bind → check shows closed. Single + range. |
| `HalfCloseTests.cs` | Stdin-EOF → write-shutdown: send a "request", get a "response", verify the listener saw EOF on its read |
| `TimeoutTests.cs` | `--check` against a non-routable address fires the timeout path. Connect timeout. |
| `CancellationTests.cs` | Trigger cancellation mid-relay, verify clean shutdown |

### Cross-platform notes

- **Privileged-port test**: skip when not running as admin (CI runners).
- **Loopback IPv6**: gate behind a "loopback IPv6 available" check; some CI containers lack `::1`.
- **TLS tests**: mock `SslStream` rather than generate real certs in CI.

### What we don't test

- Real internet endpoints (flaky, slow, ethics)
- Cross-network UDP
- Privileged-port binding (requires admin in CI)

### Coverage target

~60-80 tests total. WhoHolds has ~47, Schedule has 119; nc fits in the middle.

## Out-of-Scope but Relevant for Future

- Multi-connection listen mode (concurrent serve)
- TLS for listen mode (server cert)
- Hex dump (`-o`)
- Source-port binding (`-p`)
- Exec on connect (`-e`) — likely never; security minefield
- SOCKS proxy support
- Port forwarding (that's `socat`)
- DTLS for UDP
- `--parallel N` to override the 32-port concurrent default in check mode
